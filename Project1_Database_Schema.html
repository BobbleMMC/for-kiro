📋 1. KIRISH (Introduction)

Ushbu hujjat Minecraft Mod Generator ilovasining ma'lumotlar bazasi tuzilmasini batafsil tavsiflaydi. SQLite relational database ishlatiladi loyiha ma'lumotlarini, mod komponentlarini va foydalanuvchi sozlamalarini saqlash uchun.

⚠️ Muhim: Ushbu schema asosiy ilovaning barcha xususiyatlarini qo'llab-quvvatlay olgan holda mo'ljallangan. Kelajakda yangi jadvallar qo'shilishi mumkin.

💻 4. TO'LIQ SQL SKRIPTI
-- Minecraft Mod Generator - Database Schema
-- SQLite Format

-- PROJECTS JADVALI
CREATE TABLE projects (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL UNIQUE,
    description TEXT,
    minecraft_version TEXT NOT NULL,
    mod_loader TEXT NOT NULL,
    author TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- BLOCKS JADVALI
CREATE TABLE blocks (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    project_id INTEGER NOT NULL,
    block_name TEXT NOT NULL,
    display_name TEXT,
    hardness REAL NOT NULL,
    resistance REAL NOT NULL,
    texture_url TEXT,
    color_hex TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (project_id) REFERENCES projects(id)
        ON DELETE CASCADE
);

-- ITEMS JADVALI
CREATE TABLE items (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    project_id INTEGER NOT NULL,
    item_name TEXT NOT NULL,
    display_name TEXT,
    max_stack INTEGER NOT NULL,
    rarity TEXT,
    texture_url TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (project_id) REFERENCES projects(id)
        ON DELETE CASCADE
);

-- CRAFTING_RECIPES JADVALI
CREATE TABLE crafting_recipes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    project_id INTEGER NOT NULL,
    recipe_name TEXT NOT NULL,
    result_item_id INTEGER NOT NULL,
    recipe_json TEXT NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (project_id) REFERENCES projects(id)
        ON DELETE CASCADE,
    FOREIGN KEY (result_item_id) REFERENCES items(id)
        ON DELETE CASCADE
);

-- INDEKSLAR (Tezlik uchun)
CREATE INDEX idx_projects_name ON projects(name);
CREATE INDEX idx_blocks_project_id ON blocks(project_id);
CREATE INDEX idx_blocks_name ON blocks(block_name);
CREATE INDEX idx_items_project_id ON items(project_id);
CREATE INDEX idx_items_name ON items(item_name);
CREATE INDEX idx_recipes_project_id ON crafting_recipes(project_id);
CREATE INDEX idx_recipes_result_item_id ON crafting_recipes(result_item_id);
⚡ 5. INDEKSLAR VA OPTIMIZATSIYA

Qo'shimcha jadvallar bo'yicha tez qidiruv uchun indekslar yaratilgan:

  • idx_projects_name: Proyekt nomini qidirish uchun
  • idx_blocks_project_id: Proyektning barcha bloklarini olish uchun
  • idx_blocks_name: Blok nomini qidirish uchun
  • idx_items_project_id: Proyektning barcha itemlarini olish uchun
  • idx_items_name: Item nomini qidirish uchun
  • idx_recipes_project_id: Proyektning barcha retseptlarini olish uchun
  • idx_recipes_result_item_id: Item'dan qaysi retseptlar yasalganini bilish uchun
📝 6. FOYDALI QUERY MISOLLARI
-- 1. Barcha proyektlarni olish
SELECT * FROM projects;

-- 2. Bitta proyektning barcha bloklarini olish
SELECT b.* FROM blocks b
WHERE b.project_id = 1;

-- 3. Proyekt bilan bir tez ma'lumot olish
SELECT p.name, COUNT(b.id) as block_count,
       COUNT(i.id) as item_count
FROM projects p
LEFT JOIN blocks b ON p.id = b.project_id
LEFT JOIN items i ON p.id = i.project_id
GROUP BY p.id;

-- 4. Retsepti itemga bilanorlatirish
SELECT cr.recipe_name, i.display_name, cr.recipe_json
FROM crafting_recipes cr
JOIN items i ON cr.result_item_id = i.id
WHERE cr.project_id = 1;

-- 5. Blok va item'ni o'chirishda qayde
DELETE FROM blocks WHERE project_id = 1;
-- CASCADE orqali avtomatik aloqali bo'lmagan
